## 1. 代付訂單 → 退款申請 (Pay Order → Return Apply)

### 狀態
🟡 待確認需求

### 業務場景
```
代付訂單建立 → 自動產生退款申請 → 後台審核 → 執行出款
```

### 預期流程
1. API 收到代付請求 (amount=2000)
2. 自動組合商品湊金額
3. 建立代付訂單 (oms_order)
4. 自動建立退款申請 (oms_order_return_apply)
5. 後台審核退款申請
6. 確認後執行付款
